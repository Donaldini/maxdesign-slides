Block, inline and inline-block

In CSS2.1 the visual formatting model is used to describe types of boxes, their dimensions and how they are positioned.

We are going to have a look at the differences between three types of boxes - block, inline and inline-block.

We'll be focussing on differences in their appearance, width, height, margin, padding, border and overflow.

Block-level elements

Appearance: Block-level elements form visually a blocks, they flow down the page in normal flow.

Width: By default, block-level elements stretch to the width of their containing block or the width of the viewport. Their width can be sized but cannot collapse.

Height: By default, block-level elements heights collapse. Their height can be sized but not stretched.

Margins: Can be applied to all sides of block-level elements. This margin affects elements on any side of the block-level element.

[diagram]

Top and bottom margins will collapse when in contact with other block-level elements above or below.

[diagram]

Padding: Can be applied to all sides of block-level elements. This padding affects elements on any side of the block-level element.

Padding is added to the overall width/height of the block-level element.

[diagram]

Borders: Can be applied to all sides of block-level elements. This border affects elements on any side of the block-level element.

Border width is added to the overall width/height of the blocklevel element.

[diagram]

Overflow: Overflow can be applied to block-level elements. The default is overflow value is "visible".

Inline elements

Appearance: Inline elements are distributed in lines, they are not formatted visually as blocks, like block-level elements.

[diagram]

Width: By default, the width of inline elements is collapsed. The width cannot be sized or stretched.

[diagram]

Height: By default, the height of inline elements is collapsed. The height cannot be sized or stretched.

[diagram]

Margins: Can be applied to all sides but these margins only affects content on left and right sides.

[diagram]

Padding: Can be applied to all sides but this padding only affects content on left and right sides.

[diagram]

Border: Can be applied to all sides but these borders only affects content on left and right sides.

[diagram]

Overflow: Overflow does not apply to inline elements.

Inline-block elements

Appearance: Inline-block elements appear within lines of text like inline elements.

[diagram]

Width: By default, inline-block elements width collapses. The width can be sized or stretched.

[diagram]

Height: By default, inline-block element height collapses. The height can be sized but not stretched.

[diagram]

Margin: Can be applied to all sides of inline-block elements. This margin affects elements on all sides of the block-level element.

[diagram]

Padding: Can be applied to all sides of inline-block elements. This padding affects elements on all sides of the inline-block element.

[diagram]

Border: Can be applied to all sides of inline-block elements. This border affects elements on all sides of the inline-block element.

[diagram]

Overflow: Can be applied to inline-block elements.
