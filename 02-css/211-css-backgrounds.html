Before we dive into how CSS works, we are going to look at a range of terms that are commonly used when defining aspects of CSS.

HTML elements

HTML elements are the building blocks of HTML markup. They help us define the content and structure of web pages.

HTML elements generally consist of a start tag (or opening tag) and an end tag (or closing tag).

<p>
  This is a paragraph element
</p>

Void elements have no content and no end tag.

They can be written in two different ways depending on the doctype you use - with or without the trailing forward slash.

<!-- HTML 4.01 markup -->
<br>
<hr>
<img>
<input>
<link>
<meta>

<!-- XHTML 1.0 markup -->
<br />
<hr />
<img />
<input />
<link />
<meta />

Using the HTML 5 doctype, you can use either of these methods.

HTML attributes

HTML attributes are used to assign additional properties to HTML elements.

<p class="intro">
  Some text
</p>

Attributes generally consist of an attribute and an attribute value.

<!-- Attribute highlighted -->
<p class="intro">
  Some text
</p>

<!-- Value highlighted -->
<p class="intro">
  Some text
</p>

Attributes must appear in the start tag (or within the tag in the case of void elements)

<!-- Inside the start tag -->
<p class="intro">
  Some text
</p>

<!-- Inside void elements -->
<hr class="special">

There can be any number of attributes applied to an HTML element.

<!-- Three attributes in an element -->
<input
  id="email"
  name="email"
  type="text"
>

Attributes must be separated by spaces and can appear in any order.

Attribute values can be placed inside double quotes, single quotes or even no quotes at all.

<!-- Double quotes -->
<p class="intro">
  Some text
</p>

<!-- Single quotes -->
<p class='intro'>
  Some text
</p>

<!-- No quotes -->
<p class=intro>
  Some text
</p>

Some attributes allow multiple values inside the one attribute.

<!-- Two attribute -->
<p class="intro highlight">
  Some text
</p>

For attributes with more than one value, these values must be placed inside single or double quotes. They cannot be left unquoted.

<!-- Double quotes -->
<p class="intro highlight">
  Some text
</p>

<!-- Single quotes -->
<p class='intro highlight'>
  Some text
</p>

Some attributes do not require a value in HTML 5. These are referred to as “boolean” attributes.

<input type="radio" checked />
<input type="text" disabled />
<input type="email" multiple />
<input type="text" readonly />
<option selected> ... </option>

The document tree

All HTML documents are trees. Document trees are made from HTML elements. The document tree is just like your family tree.

An ancestor refers to any element that is connected but further up the document tree.

In the following example, the <body> element is the ancestor of the <ul> and <li> elements.

A descendant refers to any element that is connected but lower down the document tree.

In the following example, the <ul> and <li> elements are descendants of the <body> element.

A parent is an element that is connected and directly above an element in the document tree.

In the following example, the <body> element is the parent of the <ul> element.

A child is an element that is connected and directly below an element in the document tree.

In the following example, the <ul> element is the child element to the <body> element.

A sibling is an element that shares the same parent as another element.

In the following example, the <li> elements are all siblings as they share the same parent, the <ul> element.

At any time an element could be many things - parent, child, sibling, descendant, ancestor.

CSS uses the document tree to select and style elements within the document.

Block-level elements

Block-level elements are elements that are formatted visually as blocks.

Block-level elements include elements such as: 

<article> ... </article>
<aside> ... </aside>
<blockquote> ... </blockquote>
<div> ... </div>
<figure> ... </figure>
<footer> ... </footer>
<form> ... </form>
<h1> ... </h1>
<header> ... </header>
<p> ... </p>
<section> ... </section>

Inline elements

Inline elements are elements that do not form new blocks of content; the content is distributed in lines.

Inline elements include elements such as: 

<a> ... </a>
<abbr> ... </abbr>
<b> ... </b>
<cite> ... </cite>
<code> ... </code>
<em> ... </em>
<i> ... </i>
<span> ... </span>
<strong> ... </strong>
<sub> ... </sub>
<sup> ... </sup>

Containing boxes

From a CSS perspective, all HTML elements - regardless of whether they are block or inline elements - are just a series of boxes.

A containing box is a box that contains other boxes - descendant boxes.

Line boxes

Line boxes wrap around lines of text inside block level elements. They are defined by the highest and lowest points of any inline boxes that are part of the line.

Anonymous boxes

Anonymous boxes are pieces of text inside block level elements that are not made up of inline or block level elements.

User agents

The term "user agent" was coined in the early days of the Internet. Back then, the Internet could only be navigated using the keyboard.

Later, tools were developed to be the users “agent”, acting on the user’s behalf so that the user didn't have to understand the cryptic commands in order to retrieve information.

Today, nearly everyone uses a web browser as their user agent. Common web browsers include Firefox, Chrome, Safari and Opera.

However, user agents can also include media players, search engine bots and assistive technologies (software to assist people with disabilities).

The viewport

The viewport is the full extent of the viewing area within a browser window.
